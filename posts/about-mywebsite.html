聊一聊我的个人网站

2025年08月28日
ON THIS PAGE

当我接触电脑时，第一次看到 QQ 空间，就感觉很好看，关键可以表达自己，每个人可以有自己风格，那时候的 QQ 空间很火，随着 QQ 黄钻的到来，很多东西都是收费，对于一个学生来说这太奢侈了，那时候我就想找一个不要钱的平台让自己设计网站。

机缘巧合下，我成为了一名程序员，那时候还没有细分前后端，只有一个 JavaWeb 软件工程师，里面有我感兴趣的界面，奈何老师基本就是照搬图书。况且那时候上学也是以打游戏为主，很少看到圈子人玩个人网站的。

直到 2017 年，互联网细分出来一个职业，就是前端（可能更早，但是我认识前端的时候就是此时），感觉就是为了自己量身打造，我想成为一名前端，奈何那时候技术纯属半吊子，经历了半年才有一家公司愿意接受我这个前端，也是在这里我的技术突飞猛进（回过头发现还是半吊子，当初以为自己很牛），再后来越学越觉得自己很渺小，感觉前端知识好多啊，真不知道那些大佬如何学习。那时候脑子一根筋，总想找个大佬带我，后来想到了一个偏门方法来寻找大佬，那就是换工作，我知道大佬基本在大厂，所以那时候我就幻想自己进了大厂，有人带着，可是我却忘记了现实，就是前提得先进去大厂，事实证明，异想天开怎么可能提升技术呢，所以一直徘徊在小公司。

我知道学的知识不够，但是圈子里都说我已经厉害，只有我知道我压根就没入门。也许你看到了会问，你完全可以自学啊，前面的我说了，当时就是一根筋，想有个大佬，这样可以少走很多弯路，我开始尝试逛论坛，去 Github 寻找优秀代码学习，但是等我开始接触的时候，发现另外一个问题，那就是我的英文太薄弱了，我借助了 Google 翻译，但是依旧很麻烦，我这才认识到上学时真要好好学英文，我也尝试每日背诵多少个单词，但是基本半途而废。

也就是在这期间，我知道了 Github 可以搭建自己的网站，这对我来说是一个激动的消息。我开始尝试在 Github 搭建属于自己的网站，但是紧跟着就犯难了，我按照博客的标准去寻找模板，千篇一律的样式我不喜欢，而且对于我来说，半吊子的能力也是一大阻碍，我只能去找那些开源的网站代码搬运，但基本没有发现令我满意的。这期间，我开始了学习 vue，也是闲来无事看到尤大的网站很简洁，况且源码也很简单，我直接复制网页就可以了，再往后技术也上来一层，我的心思又活跃起来，就在网络上找好看的网站模板，但是翻来覆去就是那些很古板的，我想弄个简约但不简单的网站，找了很久没找到，又放弃了。

直到一位叫做 Anthony Fu，大佬很年轻，关键很牛逼，他的网站就是我一直在寻找，关键还是开源的，我赶紧搬运部署，那时候还赶紧发了人生正式的第一篇写作，再后来因为工作反而没有精力去打理网站。

这一下就过了半年之久，此时前端技术已经到了百花齐放，技术种类很多，我在抖音上刷了很多有用的知识，我迫切的需要一个地方记录，显然我的网站是个好地方，我想用自己的技术重构它，我开始纠结于技术选型，开始幻想一个好看实用完美网站，完美就意味着要求的多，我是选择 Vue 还是 React，亦或者 Lit，还尝试了原生JS，既想用新技术，又想兼容旧浏览器，我还想加个动画，还想加入 seo 的支持，还有国际化，当然也要支持在线编辑代码，反正就是光想不做。我意识到这样不行，最终我决定尝试原生 JS，发现能力果然还是不行。

我想到了之前看中的一个 vite 插件用于 ssr，是 vite-plugin-ssr，新命名为 vike，查看官网文档，作者花大经历去宣传 Vike + React 或者 Vike + Vue，也许写惯了 Vue，这次我选择了 Vike + React，希望体验下 React 的 TSX 开发，提示肯定比 Vue 要舒服的多，但是也仅限很舒服，因为习惯了 Vue 自带的 style 标签，我发现 React 处理样式的方案太多，对比很多 css-in-js 方案，最后选择了 vanilla-extract，因为只有它可以在不配置的情况下，让别人无法通过 Chrome DevTools 看出网站使用了哪种技术，满足了我的特殊癖好。

vanilla-extract 书写提示很舒服，但我真的感觉很别扭，就是感觉反人类。关键这中间，因为我尝试了大量的新技术，我需要看的文档成倍增加，它们对国人没那么友好，我的英文能力实在一塌糊涂，我发现自己似乎一直在做舍本逐末的事。潜意识里，我希望通过这个网站来加深 React 技术，加深 ssr 技术，学习最新的 css-in-js 技术，还有 mdx 技术，还有各类动画，我渴望要学的东西太多了，但是我发现，我并不是一个聪明的人。

回过头来，我发现有一个现成的技术，那就是 vitepress，我开始尝试静下心去阅读官方文档，也尝试自己制作组件，我发现 vitepress 已经在底层帮我处理了关于 markdown 的一切，而且还加了自己的特性，况且依旧是 Vue，我很熟悉。虽然还有点小问题，就是个 VsCode 的 Vue 插件，但是整体看来问题不大。

这次新尝试，我借鉴了 Sidey - Jekyll theme，依旧延续黑白灰，看上去像一份报纸，色彩盘取自 @radix-ui/colors，另外我直接适配中文，我不再考虑什么所谓的国际化，本来英语就是个半调子，估计也没几个外国人会看到，还不如把中文网站做好，当然，我加入了一个小小的彩蛋，那就是键盘事件的样式 :focus-visible 用了明亮的蓝色，它看上去很活泼（可以按 tab 键看到），这在别人的网站很少看到。

当然现在整体已经完成，还有细节需要完善，比如适配手机，比如首页内容，比如 Aria，还有字体大小归类，还有移除不必要的色彩，还有字体设计等等，当然中间我要是看到啥好看的设计还是要加进来的😄。
